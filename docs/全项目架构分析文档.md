## 1. 系统概览

### 1.1 项目定位

A 股盘中实时量化监控与**信号呈现**系统（不做自动交易）。

```
单一数据源 (tqcenter) → QuestDB 单库 → 16 个策略插件 → 飞书三通道推送
```

### 1.2 核心数据流

```
tqcenter (通达信 COM 组件)
    │
    ├── c1_pricevol    → qd_pricevol (全场价量)
    ├── c2_snapshot   → qd_stock/sector/index_snapshot (快照)
    ├── c3_more_info   → qd_stock/sector/index_daily + qd_stock_intraday
    ├── c4_kline      → qd_kline_1m + qd_kline_5m
    ├── c5_mapping    → qd_sector_* (板块映射)
    └── c6_lhb       → qd_lhb_detail + qd_lhb_broker (龙虎榜)
            │
            ▼
    QuestDB (存储层)
            │
            ▼
    compute (k1-k5)
    - k1_indicators   → qd_indicators (技术指标)
    - k2_signals      → qd_signals (原子信号)
    - k3_sentiment    → qd_sentiment_snapshot_min + qd_sentiment_event_log
    - k4_sentiment    → qd_sentiment_deep (深度情绪)
    - k5_kline_synth  → qd_kline_5m (合成)
            │
            ▼
    strategy (策略层)
    - StrategyRegistry 插件遍历
    - 16 个策略插件 (p01-p25)
            │
            ▼
    feishu (通知层)
    - push (卡片推送)
    - sheet_writer (电子表格)
    - bitable_writer (多维表格)
```

### 2.2 scheduler.py 主调度器

**核心逻辑**: 基于交易时钟的软实时调度，不使用 cron 表达式。

| 时间 | 阶段 | 行为 |
|------|------|------|
| ~09:07 | pre_market | 提前 8 分钟启动 auction_monitor |
| 09:15 | auction | auction_monitor subprocess 运行 |
| ~09:22 | pre_market | 停 auction, 启动 intraday_loop |
| 09:25-09:30 | pre_market | daily_init.run() (一次性) |
| 09:30 | morning | intraday_loop subprocess 运行 |
| ~12:52 | lunch | 提前 8 分钟启动 intraday 下午 |
| 13:00 | afternoon | intraday_loop 继续运行 |
| ~14:49 | pre_close | 提前 8 分钟启动 auction |
| 14:57 | pre_close | auction_monitor subprocess 运行 |
| 15:00 | closed | intraday_loop 退出 |
| 15:05 | closed | daily_close + daily_summary |
| 16:00 | closed | verify_tables (DDL 校验) |

## 4. 数据计算层 (compute/)

### 4.1 k1 技术指标 (compute/k1_indicators.py)

**频率**: 60s/轮

**关键参数**:
```python
MACD_FAST = 12      # EMA12
MACD_SLOW = 26      # EMA26
MACD_SIGNAL = 9     # DEA 平滑
BOLL_N = 20         # 布林周期
BOLL_K = 2.0        # 布林倍数
PRESS_N = 20        # 压力位根数
KLINE_ROWS_PER_CODE = 30  # 每 code 取 30 根 5m K线
```

**计算公式**:
```
DIF = EMA(close, 12) - EMA(close, 26)
DEA = EMA(DIF, 9)
HIST = (DIF - DEA) * 2

BOLL_MID = MA(close, 20)
BOLL_STD = STD(close, 20)
BOLL_UPPER = BOLL_MID + 2 * BOLL_STD
BOLL_LOWER = BOLL_MID - 2 * BOLL_STD

PRESSURE_HIGH = MAX(high, 20根)
SUPPORT_LOW = MIN(low, 20根)

MA5 = MA(close, 5)
MA10 = MA(close, 10)
MA20 = MA(close, 20)
```

### 4.2 k2 原子信号 (compute/k2_signals.py)

**频率**: 60s/轮

| 信号类型 | 判定条件 |
|---------|---------|
| golden_cross | `prev_dif <= prev_dea AND cur_dif > cur_dea` |
| death_cross | `prev_dif >= prev_dea AND cur_dif < cur_dea` |
| break_pressure | `prev_close < prev_pressure_high AND cur_close >= cur_pressure_high` |
| break_support | `prev_close > prev_support_low AND cur_close <= cur_support_low` |

**信号评分**: 固定 `1.0`

### 4.3 k3 大盘情绪 (compute/k3_sentiment.py)

**频率**: 60s/轮

**阈值集中管理 (TH 类)**:

```python
# 情绪评级 4 分量阈值
EMOTION_ZT = (30, 60, 100, 150)       # 涨停数 → 冰点/低迷/中性/活跃/过热
EMOTION_FBL = (60.0, 70.0, 80.0, 90.0) # 封板率%
EMOTION_LB = (3, 4, 6, 9)             # 最高连板
EMOTION_UDR = (0.5, 1.0, 2.0, 3.0)    # 涨跌比

# 涨跌停判定
SEAL_AMO_POS = 0           # FCAmo > 0 = 涨停
NEAR_ZT_RATIO = 0.999     # Max >= ZTPrice * 0.999 = 曾触涨停

# 封板率公式
FBL = zt / (zt + break) * 100

# 涨跌比公式
UDR = up / down  (cap at 99.0)
```

**情绪评级算法**: 4 分量各评 1 档，取最差档 (保守)

### 4.4 k4 深度情绪 (compute/k4_sentiment.py)

**频率**: 5min/轮 (独立进程 scheduler 调度)

**恐慌/贪婪指数 (PG_INDEX) 权重**:

| 因子 | 最小值 | 最大值 | 权重 |
|------|--------|--------|------|
| zt_cnt (涨停热度) | 0 | 150 | 0.20 |
| fbl (封板率%) | 0.0 | 90.0 | 0.20 |
| udr (涨跌比) | 0.0 | 4.0 | 0.20 |
| max_lb (最高连板) | 0 | 10 | 0.15 |
| main_net (主力净流) | -5e9 | 5e9 | 0.15 |
| pressure_diff (5档压力差) | -5e7 | 5e7 | 0.10 |

**PG 信号档位**:
```
0-25: 恐慌  |  25-45: 恐惧  |  45-65: 中性  |  65-85: 贪婪  |  85-100: 狂热
```

### 4.5 k5 K线合成 (compute/k5_kline_synth.py)

**功能**: 将 1m K线合成当日 5m K线

**入库条件**:
- 目标表: `qd_kline_5m`
- 只合成 focus 池 (qd_stock_snapshot 是 focus 池高频)

### 5.3 P01 涨停打板评分公式

```python
score = min(100.0,
    50.0
    + fcamo / 1e4
    + cnt * 3
    + lianb * 2
    + lb_rate * 0.3
    + (red_rate - 70) * 0.5
)
```

- `fcamo`: 封单额
- `cnt`: 板块涨停数
- `lianb`: 量比 (fLianB)
- `lb_rate`: 连板率 (gp40_lb_rate)
- `red_rate`: 次日红盘率 (gp39_next_red_rate)

| 表类型 | 分区策略 | 理由 |

| 日频快照 (pricevol/snapshot/kline) | DAY | 高频数据，按日分区便于清理 |
