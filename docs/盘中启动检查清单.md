# 明日盘中启动检查清单

## 交易日前必做（预计 3 分钟）

### 1. 启动 QuestDB（30 秒）

方法一：Explorer 双击 `K:\QuestDB\bin\questdb.exe`
方法二：双击 `K:\QuestDB_test\start_questdb.bat`

验证：
```cmd
tasklist | findstr java        # 应有 1 个 java.exe
curl http://127.0.0.1:9000     # HTTP Console 就绪
```

### 2. 启动调度器（全自动）

调度器（`runner/scheduler.py`）管理所有子进程的生命周期，包括竞价、盘中、盘后脚本。

```cmd
cd /d K:\QuestDB_test
python runner/scheduler.py
```

调度器会自动：
- **09:15** → 启动 `auction_monitor`（竞价监控，09:15-09:25）
- **09:30** → 启动 `intraday_loop`（盘中主循环，09:30-15:00）
- **15:00** → 启动 `daily_summary` + `daily_close`（盘后更新）
- **全程** → `overseer` 监工每 15 秒心跳检查各进程状态

### 盘前加项：运行 daily_init（约 2 分钟）

```cmd
cd /d K:\QuestDB_test
python runner/daily_init.py
```

验证：
```cmd
python -c "from lib.qdb import connect; import pandas as pd; print(pd.read_sql_query('SELECT max(trade_date) FROM qd_stock_daily', connect()))"
```

### 确认 parquet 备份在工作（验证性写入）

盘中第一个批次写入后：

当日 parquet 文件应出现（如 `2026-07-08.parquet`）。

## 故障恢复

### 情况 A：QuestDB 挂了

1. 关闭所有 Python 脚本
2. 双击 `K:\QuestDB_test\recover_questdb.bat`（一键恢复）
3. 等待脚本提示"恢复完成"
4. 重启 runner 脚本

### 情况 B：只有 parquet 备份但 QuestDB 全丢

1. 双击 `recover_questdb.bat`
2. DuckDB 临时查询：
```sql
SELECT * FROM 'D:\dbshujubeifen\qd_pricevol\2026-07-08.parquet';
```

### 情况 C：开机找不到任何表（tables.d 被覆盖）

直接用 `recover_questdb.bat` 一键恢复。
